Exhibition Installation — Odysee Live

A human-height interactive installation that renders Odysee's multi-agent architecture as a physical experience. Visitors speak; the agents activate live on screen; the answer comes back in voice.

Venue HIT Holon
Dates Aug 6–19, 2026
Interaction ~90 sec
Target latency ~2.7 sec
01

The Concept

The installation is not a chatbot demo. It is a live visualization of the architecture — the thesis itself rendered as experience. Odysee is the system; Tico, the primary agent, is its voice. At the screen's center, the Orchestrator glows in gold; around it, ten specialized agents wait. When a visitor speaks to Tico, the edges between agents light up in real time, revealing which agent handles what.

There is no anthropomorphic avatar. The choice is deliberate: the thesis contribution is the multi-agent architecture, not the conversational UI. A talking face would obscure the agents; a living constellation reveals them.

Visitor Experience

01

Approach

Visitor approaches the screen. The camera detects presence; the screen transitions from idle to listening state.

02

Voice Prompt

The visitor speaks naturally: "Plan me a week in Portugal." The shotgun mic captures the speech; VAD detects end-of-utterance.

03

Tico Responds

Tico, the voice of Odysee, wakes up. The Orchestrator glows first. Gold lines extend to the Inspiration Agent, then to the Itinerary Agent. A2A messages flow as pulses along the edges.

Live Response

The live transcript appears below in both languages. ElevenLabs voices the response through the speaker while the next sentence is still being generated — target latency: 2.7 sec to first audio.

Architecture Reveal

When the conversation ends, the constellation remains lit for a moment — the visitor leaves with a visual understanding of how the system worked for them.

03

Technical Architecture

The pipeline from microphone to voice is a five-layer chain, each with a defined latency budget. The core principle: Claude's streaming allows ElevenLabs to begin playing the response before it finishes generating, saving ~2 seconds of perceived wait.

Layer Component Technology Latency budget
Input Audio capture USB shotgun mic, cardioid —
Local VAD + noise Silero VAD, RNNoise 200 ms
Cloud STT OpenAI Whisper API 600 ms
Cloud Orchestration + response Claude + LangGraph · 11 agents · streaming 1500 ms
Cloud TTS ElevenLabs streaming 400 ms
Output Visualization + voice React + WebSocket · Bluetooth speaker —
Total to first audio ~2.7 sec

05

Risks & Fallbacks

Risk Impact Mitigation
Network failure Critical Wired Ethernet + 4G LTE backup + pre-recorded demo mode
Exhibition noise High Shotgun cardioid + VAD threshold + push-to-talk button
API latency spike / outage High 5-second timeout + 'thinking...' indicator + generic fallback
Compute crash after long run High Auto-restart script + daily manual reset before opening
Unsupported language Medium Language detection → English response with apology
